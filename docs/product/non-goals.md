# SafeHire — Non-Goals

**Last Updated:** 2026-03-22

Things SafeHire must NOT become, attempt, or drift toward.

---

## Product Non-Goals

1. **LinkedIn for workers** — no professional networking, no connections, no feed, no endorsement walls
2. **Resume product** — workers are not writing about themselves; trust is verified externally
3. **Profile vanity** — no banner photos, no headline editors, no "about me" sections
4. **Generic SaaS dashboard** — every surface must serve a specific actor with a specific task
5. **Blacklist system** — incidents are reviewed, time-bound, and appealable
6. **Surveillance platform** — workers own their data, consent is required for access
7. **UUID-driven UX** — no user should ever need to know, copy, or paste a UUID
8. **Freeform access requests** — "why do you need access?" paragraph boxes are not acceptable
9. **Feature sprawl** — no chat, no AI matching, no payments, no insurance this sprint
10. **Over-engineered abstractions** — no component libraries, design systems, or refactors without UX benefit

## Technical Non-Goals This Sprint

- Full DigiLocker/Aadhaar integration
- Razorpay/payment processing
- Google Maps integration
- Mobile app changes
- Admin panel redesign
- Multi-language support
- Real KYC verification flow
- Background check integrations

## Tone Non-Goals

- Never condescending toward workers
- Never suspicious or surveillance-like in language
- Never corporate or enterprise-feeling
- Never startup-buzzword-heavy ("synergy," "leverage," "optimize")
